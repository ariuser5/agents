# Clarify

Use when a missing requirement or consequential choice blocks safe execution.

1. Separate facts already discoverable locally from decisions only the user can
   make. Research the former before asking about the latter.
2. Reduce the issue to one decision whenever possible.
3. State why the answer changes implementation, behavior, risk, or scope.
4. Recommend the safest option consistent with the request.
5. Offer no more than two other materially different options.
6. Continue reversible read-only work while waiting when useful.

Do not ask about routine implementation details, naming that follows local
conventions, or choices with an obvious reversible default.
