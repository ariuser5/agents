# Research

Use `rules/research-policy.md` as the controlling policy.

1. Write down the decision or unknown the research must resolve.
2. Check active ticket context and exact user-provided paths first.
3. Batch the smallest set of independent reads likely to answer it.
4. Compare findings against local conventions and authoritative documentation.
5. Stop once the evidence supports an action and verification plan.
6. Summarize only findings that affect the next step; distinguish fact,
   inference, and unresolved uncertainty.

Avoid repository-wide scanning when a symbol, path, test, configuration key, or
error message can narrow the search.
